## 1. Google Chrome Developer Tools Shortcuts

Because Chrome is the dominant [browser for web development](/blog/chrome-developer-tools-explained/), memorizing the Chrome developer tools keyboard shortcuts is mandatory for frontend engineers.

### Opening & Closing Chrome DevTools

| Action | Windows / Linux | Mac (macOS) |
| :--- | :--- | :--- |
| **Open Developer Tools** | `F12` or `Ctrl + Shift + I` | `Cmd + Option + I` |
| **Open Console Panel Directly** | `Ctrl + Shift + J` | `Cmd + Option + J` |
| **Inspect an Element (Toggle Picker)** | `Ctrl + Shift + C` | `Cmd + Option + C` |
| **Close DevTools** | `F12` | `Cmd + Option + I` |

### Navigating Chrome DevTools

| Action | Windows / Linux | Mac (macOS) |
| :--- | :--- | :--- |
| **Next Panel (Tab right)** | `Ctrl + ]` | `Cmd + ]` |
| **Previous Panel (Tab left)** | `Ctrl + [` | `Cmd + [` |
| **Toggle Device Device Toolbar (Mobile)** | `Ctrl + Shift + M` | `Cmd + Shift + M` |
| **Search in All Files** | `Ctrl + Shift + F` | `Cmd + Option + F` |

*Pro Tip: While searching files in Chrome is useful, if you need to rapidly format or manipulate a massive JSON API response you found in the Network tab, paste it into our offline [**JSON Formatter**](/tools/json-to-code-generator/) instead of struggling with Chrome's tiny text viewer.*

## 2. Mozilla Firefox Developer Tools Shortcuts

Firefox Developer Edition uses an almost identical shortcut schema to Chrome, making it very easy to switch between the two browsers for cross-browser testing.

### Opening & Closing Firefox DevTools

| Action | Windows / Linux | Mac (macOS) |
| :--- | :--- | :--- |
| **Open Web Developer Tools** | `F12` or `Ctrl + Shift + I` | `Cmd + Option + I` |
| **Open Web Console Directly** | `Ctrl + Shift + K` | `Cmd + Option + K` |
| **Open Style Editor** | `Shift + F7` | `Shift + F7` |
| **Open Network Monitor** | `Ctrl + Shift + E` | `Cmd + Option + E` |

### Navigating Firefox DevTools

| Action | Windows / Linux | Mac (macOS) |
| :--- | :--- | :--- |
| **Toggle Responsive Design Mode** | `Ctrl + Shift + M` | `Cmd + Option + M` |

## 3. Apple Safari Developer Tools Shortcuts

If you are developing on a Mac, you must occasionally use Safari to ensure WebKit compatibility. However, Safari requires a specific step before the shortcuts will work.

### How to Enable Developer Tools in Safari
By default, Apple hides the Developer menu in Safari. To enable it:
1. Open Safari and click **Safari** in the top menu bar > **Settings** (or Preferences).
2. Go to the **Advanced** tab.
3. Check the box at the very bottom: **"Show Develop menu in menu bar"**.

### Opening & Closing Safari DevTools

Once enabled, you can use these shortcuts:

| Action | Mac (macOS) |
| :--- | :--- |
| **Open Developer Tools Safari** | `Cmd + Option + I` |
| **Open Error Console Directly** | `Cmd + Option + C` |
| **Show Page Source** | `Cmd + Option + U` |
| **Enter Responsive Design Mode** | `Cmd + Ctrl + R` |
